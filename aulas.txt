Tipos e variáveis

Tipos de dados:
- No java existem palavras reservadas para representar os tipos de dados básicos que precisam ser manipulados para construir programas
- Estes tipos de dados são conhecidos como tipos primitivos (primitive types):
    - byte (1 byte), short (2 bytes), int (4 bytes), long (8 bytes), *partes fracionárias:* float (4 bytes), double (8 bytes) - garantia de precisões, boolean, char
    - Não são objetovs, são valores brutos
    - Armazenados diretamente na pilha de memória (memory stack)

Declaração de variáveis
- Variável é a identificação de um espaço em memória utilizado pelo nosso programa
- A variável é composta por: tipo de dados + identificação + valor atribuído
- Um double é com . para decimal: double salarioMinimo = 2500.33;
- Long: long cpf 98765432109L
- float pi = 3.14F  -> Para perceber que é um float e não um double
- Java é fortemente tipado!!!!
-short numeroCurto = 1;
    int numeroNormal = numeroCurto;
    // short numeroCurto2 = numeroNormal; -> Não iria funcionar, pois o número normal é um int e o número curto é um short, e o Java não tem como adivinhar se o número normal vai caber dentro do número curto. Para resolver isso, precisamos fazer um cast, ou seja, dizer para o Java que temos a certeza que o número normal vai caber dentro do número curto. 
    short numeroCurto2 = (short) numeroNormal;

Variáveis e constantes
- Variável é uma área de memória, associada a um nome, que pode armazenar valores de um determinado tipo
- O tipo de dado define um conjunto de valores e um conjunto de operações
- Java é uma linguagem com muita rigidez de tipos
- Utilizamos uma referência (ponteiro) a um valor de memória, e esta pode ser redirecionada a outro valor (daí o nome "variável")

- Constantes são valores armazenados em memória, que não podem ser modificados depois de serem declarados
- São representadas pela palavra "final"
- São escritas em maiúsculas
- Exemplo: final double VALOR_DE_PI = 3.14;

Operadores
- Símbolos especiais que usamos para expressões/operações matemáticas
- Um programa é um conjunto de instruções lógicas que, quando executadas, produzem algum Resultado

Classificação de Operadores
- Atribuição -> símbolo de igualdade = ; é usado para definir valor inicial ou sobreescrever o valor de uma variável

Aritméticos
- Usado para realizar operações matemáticas entre valores numéricos
- + adição, - subtração, * multiplicação, / divisão
- a + quando usada em variáveis do tipo texto, procede à concatenação de texto em vez de adição

Unários
- São aplicados junto com outro operador Aritméticos
- Incrementam, decrementam, invertem valores numéricos e booleanos
- + unário de valor positivo
- - unário de valor negativo
- ++ unário de incremento de valor
- -- unário de decremento de valor
- ! unário lógico de negação

Ternário
- Operador de condição ternária é uma forma resumida para definir uma condição e escolher entre 2 valores
- É como um if normal mas escrito numa só linha
- É representado pelo símbolo ?
- <Expressão condicional> ? <Caso a condição seja true> : <Caso seja false>
- Substitui isto:
    int a,b

    a=5;
    b=6;

    String resultado = "";

    if(a==b)
        resultado = "verdadeiro";
    else 
        resultado = "falso";
    
    System.out.println(resultado);
- Por isto:
    int a,b

    a=5;
    b=6;

    String resultado = a==b ? "verdadeiro" : "falso";

Relacionais
- Avaliam a relação entre duas variáveis ou expressões
- Verificam se o operando da esquerda é ==, !=, <=, >=, <, > ao da direita e retorna um resultado em forma de boolean

- Com objetos utilizamos o equals em vez do ==. Compara conteúdos; só devemos usar o == com números

Lógicos
- Permitem criar expressões lógicas maiores a partir da junção de 2 ou + expressões
- && e, || ou

Métodos
- Todas as ações das aplicações são consideradas métodos
- Uma classe é definida por atributos e métodos:
    - Atributos são variáveis de ='s tipos e valores
    - Métodos correspondem a funções/ações ou sub-rotinas disponíveis dentro das classes (ex: acelerar, travar, ...)
- Convenções para nomenclatura de métodos:
    - Devem ser verbos
    - Seguir padrão camelCase
    - Ex: somar(), abrirConexao(), concluirProcessamento()
    - Em java não existem Métodos Globais. Todos os métodos devem ser SEMPRE definidos dentro de uma classe
- Critérios para definir métodos:
    - Qual a finalidade do método?
    - Qual o retorno esperado depois da execução do método? Caso não retorne nada, será usada a palavra void
    - Quais os parâmetros necessários para a execução do método?
    - O método tem algum risco de apresentar alguma exceção? Usar o throws Exception:
        Ex: public double dividir(int dividendo, int divisor) throws Exception (-> Ao usar 0 como divisor por exemplo)
    - Qual a visibilidade do método? Visível no mesmo pacote, em toda a aplicação, através de herança, ou só na própria classe

Scope
- É o ambiente onde uma variável pode ser acedida; em java, vai de acordo com o bloco onde foi declarada
- A variável é criada no 1º acesso à mesma e torna-se inacessível assim que o interpretador sai do bloco de execução ao qual pertence
- Numa classe os atributos (variáveis) são declarados no corpo principal da classe e podem, portanto, ser acessíveis por todos os métodos
- Se declararmos uma variável dentro de um método, o scope dela fica limitado ao corpo desse mesmo método

Scanner

Controlo de fluxo
- Habilidade de ajustar a maneira como um programa realiza as suas tarefas, por meio de:
    - Instruções especiais
    - Chamadas de comandos
- Essas tarefas podem ser executadas:
    - Seletivamente, Repetidamente, Excecionalmente

Os controles de fluxo podem classificar-se em:
- ESTRUTURAS CONDICIONAIS:
    - if-else, switch-case
- ESTRUTURAS DE REPETIÇÃO:
    - for, while, do-while
- ESTRUTURAS DE EXCEÇÕES:
    - try-catch-finally, throw

----

ESTRUTURAS CONDICIONAIS
- Possibilita a escolha de um grupo de ações e comportamentos a ser executados quando determinadas condições são/não são satisfeitas
- Pode ser simples ou composta:
    SIMPLES
    - Quando é feita uma validação de execução de fluxo apenas quando a condição é positiva
    - Ex: Início -> Sacar -> Contém saldo? -> Sim (Atualiza saldo) -> Não (Fim)
    - Quando uma condição precisa ser verdadeira para executar a ação
    - If simples

    COMPOSTA
    - Quando o nosso programa deve seguir mais do que uma jornada de execução, condicionado a uma regra de negócio
    - Ex: Início -> Resultado Escolar -> Nota >= 7? -> Sim (Imprime Aprovado) -> Não (Imprime Reprovado)
    - If - else

    ENCADEADA
    - Quando temos uma 3ª. 4ª, ..., condição para além do if-else (ou seja - if - else if - else if - else)
    - Início -> Resultado Escolar -> Nota >= 7? -> Sim (Imprime Aprovado) -> Não -> Nota >= 5 e < 7? -> Sim (Imprime Recuperação) -> Não (Imprime Reprovado) 

    TERNÁRIA
    - Podemos abreviar o nosso algoritmo com o conceito de operador Ternário
    - Ex: String resultado = nota >= 7? "Aprovado" : "Reprovado";
    - Ex: String resultado = nota >= 7? "Aprovado" : nota >= 5 && nota <7? "Recuperação" : "Reprovado";

    SWITCH
    - Compara o valor de cada caso com o da variável de forma sequencial e sempre que encontra um valor correspondente, executa o código associado ao caso
    - Para evitar que continuem a ser executadas comparações após um caso correspondente ter sido encontrado, usamos o comando "break" no fim de cada bloco
    - o break, ao ser executado, encerra a execução da estrutura onde se encontra
    - Ex: preciso imprimir uma medida de acordo com uma sigla P(pequeno), M(médio), G(grande)
        String sigla = "M";

        if(sigla == "P")
            System.out.println("PEQUENO");
        else if(sigla == "M")
            System.out.println("MEDIO");
        if(sigla == "G")
            System.out.println("GRANDE");
        else
            System.out.println("INDEFINIDO");

        ---
        com switch:
        switch (sigla) {
            case "P": {
                System.out.println("PEQUENO");
                break;
            }
            case "M": {
                System.out.println("MEDIO");
                break;
            }
            case "G": {
                System.out.println("GRANDE");
                break;
            }
            default:
                System.out.println("INDEFINIDO");
        }

ESTRUTURAS DE REPETIÇÃO
- Laços de repetição ou laços de iteração ou loops são comandos que permitem a iteração de um código (repetição de comandos)
- São representados pelas seguintes estruturas:
    - FOR
        - Em que uma variável contadora é testada/incrementada a cada iteração
        - Recebe o contador, a condição e o valor de incrementação
        - for(bloco de iniciação; expressão booleana de validação; bloco de atualização)
        
        - FOR EM ARRAYS
            - Usamos o controlo de fluxo FOR para interagir sobre arrays e coleções
            - Ex:
                String alunos[] = {"Ana", "Artur", "Bruno", "Ricardo"}

                for (int x=0; x<alunos.length; x++)
                {
                    System.out.println("O aluno no indice x=" + x + " é " + alunos[x]);
                }
        - FOR EM COLEÇÕES - FOR EACH
            - A interação é baseada nos elementos da coleção
            - for (String aluno : alunos) {
                System.out.println("Nome do aluno é: " + aluno);
            }
    
    - BREAK/CONTINUE
        - Break - Significa quebrar/interromper uma operação
        - Continue - Continua o laço, só interrompe a iteração atual
    
    - WHILE
        - Determina que enquanto uma condição for válida, o bloco de código é executado
        - Testa a condição antes de executar o código
        - Ex: while(expressão booleana de validação)

    - DO WHILE
        - Ao contrário do while, o do/while testa a condição APÓS executar o código. Será corrido pelo menos uma vez
        - Ex: System.out.println("A ligar...");

        do {
            System.out.println("Telefone a tocar...");
        } while(tocando());

Estruturas excepcionais - EXCEÇÕES
- Ao executar código em java, podem acontecer diferentes erros:
    - Erros de codificação (pelo programador)
    - Erros devido a entrada errada
    - Outros imprevistos
- Quando ocorre um erro, o java pára e gera uma mensagem de erro (lança uma exceção)
- Um erro é algo irreparável em que a app trava/é encerrada drasticamente
- Uma exceção é um comportamento/fluxo inesperado da app (tentar abrir um ficheiro que não existe, dividir um nº por 0, abrir conexão com
BD com pw errada,...) -> fluxos não previstos pela aplicação
- Um dos papéis do programador é prever certos comportamentos e fazer TRATAMENTO DE EXCEÇÕES
    - try permite definir um código a ser testado quanto a erros durante a sua execução
    - catch permite definir um bloco de código a ser executado caso ocorra erro no try
    - finally permite definir código a ser executado quer haja erro no try ou não

Hierarquia das exceções e exceções personalizadas
- Java possui várias classes que representam exceções, organizadas numa hierarquia chamada:
"checked and unchecked exceptions"
- Hierarquia de erros
         -->    throwable   <--
        |                      |
    Error                   Exception
                        |                   |    
                    RuntimeException        IOException/SQLException/MalformedURLException
                        |
            NullPointerException
            NumberFormatException
            ClassCastException
            IndexOutOfBoundsException
- Ser checked ou não checked tem a ver com a possibilidade/obrigatoriedade de corrigir ou não essas exceções

Exceções customizadas
- Podemos criar exceções baseadas em regras de negócio
- Ex: para formatar um NIF precisamos sempre de 9 digitos, caso contrário lança uma exceção "NifInvalidoException"
- Para criar a exceção:
    public class NifInvalidoException extends Exception {}
- Para usar:
    if(nif.length() != 8)
        throw new NifInvalidoException();
- Dentro de um try catch
    try{
        String nifFormatado = formatarNif("1234567885332");
        System.out.println("nifFormatado");
    } catch (NifInvalidoException e) {
        System.out.println("O nif não tem o comprimento obrigatório!");
    }
